部署靜態網站到 AWS(S3 + CloudFront + GitHub Actions)

Deployment 2026-07-06 English version
TL;DR

靜態網站放喺 private S3 bucket,前面用 CloudFront(CDN + HTTPS)經 OAC 讀取;GitHub Actions 用 OIDC(唔使長期 access key)攞臨時 credentials,merge 入 main 就自動 aws s3 sync + invalidate CloudFront cache。一次過設定好之後,以後只需 push code。

架構

GitHub repo ──(merge to main)──▶ GitHub Actions
                                     │  OIDC assume role(冇長期 key)
                                     ▼
                              aws s3 sync site/ ──▶ S3 bucket(private)
                                     │                    ▲
                                     ▼                    │ OAC(只准 CloudFront 讀)
                       cloudfront create-invalidation     │
                                                    CloudFront(HTTPS + CDN)
                                                          ▲
                                                        用戶

Step 1:開 S3 bucket

  1. S3 → Create bucket,名要全球唯一,例如 my-knowledge-base-site。
  2. Block all public access 全部保持啟用 — bucket 唔需要 public,CloudFront 會經 OAC 讀。
  3. 唔使開 static website hosting(嗰個係 public website endpoint 先要,OAC 模式用普通 REST endpoint)。

Step 2:開 CloudFront distribution(用 OAC)

  1. CloudFront → Create distribution,Origin 揀你個 S3 bucket。
  2. Origin access 揀 Origin access control settings (OAC) → create new OAC。
  3. Viewer protocol policy:Redirect HTTP to HTTPS。
  4. Default root object:index.html(唔設嘅話開 root 會 403)。
  5. 建立之後,CloudFront 會提示你 copy 一段 bucket policy 去 S3 — 貼上去(Permissions → Bucket policy),大致係咁:
{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Allow",
    "Principal": { "Service": "cloudfront.amazonaws.com" },
    "Action": "s3:GetObject",
    "Resource": "arn:aws:s3:::<BUCKET>/*",
    "Condition": {
      "StringEquals": {
        "AWS:SourceArn": "arn:aws:cloudfront::<ACCOUNT_ID>:distribution/<DIST_ID>"
      }
    }
  }]
}
要自訂 domain 嘅話:喺 distribution 加 alternate domain name,再喺 ACM(一定要 us-east-1 region)申請 certificate,最後喺 DNS 加 CNAME/alias record。

Step 3:IAM — GitHub OIDC provider + deploy role

用 OIDC 俾 GitHub Actions 直接 assume role,唔使喺 GitHub 存 AWS access key,冇 key 洩漏、rotate 嘅問題。

3a. 開 OIDC identity provider(每個 AWS account 一次)

IAM → Identity providers → Add provider:

  • Provider type:OpenID Connect
  • Provider URL:https://token.actions.githubusercontent.com
  • Audience:sts.amazonaws.com

3b. 開 deploy role

IAM → Roles → Create role → Web identity,揀啱上面個 provider。Trust policy 要限定你個 repo 同 main branch:

{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Allow",
    "Principal": {
      "Federated": "arn:aws:iam::<ACCOUNT_ID>:oidc-provider/token.actions.githubusercontent.com"
    },
    "Action": "sts:AssumeRoleWithWebIdentity",
    "Condition": {
      "StringEquals": {
        "token.actions.githubusercontent.com:aud": "sts.amazonaws.com"
      },
      "StringLike": {
        "token.actions.githubusercontent.com:sub": "repo:<GITHUB_USER>/<REPO>:ref:refs/heads/main"
      }
    }
  }]
}

Permissions policy 只俾最少權限(sync + invalidation):

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": ["s3:ListBucket"],
      "Resource": "arn:aws:s3:::<BUCKET>"
    },
    {
      "Effect": "Allow",
      "Action": ["s3:PutObject", "s3:GetObject", "s3:DeleteObject"],
      "Resource": "arn:aws:s3:::<BUCKET>/*"
    },
    {
      "Effect": "Allow",
      "Action": ["cloudfront:CreateInvalidation"],
      "Resource": "arn:aws:cloudfront::<ACCOUNT_ID>:distribution/<DIST_ID>"
    }
  ]
}

Step 4:GitHub repo secrets

Repo → Settings → Secrets and variables → Actions,加四個 secret:

Secret值
AWS_DEPLOY_ROLE_ARNStep 3b 個 role ARN,例如 arn:aws:iam::123456789012:role/kb-deploy
AWS_REGIONbucket 所在 region,例如 ap-southeast-1
S3_BUCKETbucket 名
CLOUDFRONT_DISTRIBUTION_ID例如 E1A2B3C4D5E6F7

Step 5:GitHub Actions workflow

放喺 .github/workflows/deploy.yml(呢個 repo 已經有):

name: Deploy to AWS

on:
  push:
    branches: [main]
  workflow_dispatch:

permissions:
  id-token: write   # OIDC 必須
  contents: read

concurrency:
  group: deploy
  cancel-in-progress: true

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: aws-actions/configure-aws-credentials@v4
        with:
          role-to-assume: ${{ secrets.AWS_DEPLOY_ROLE_ARN }}
          aws-region: ${{ secrets.AWS_REGION }}

      - name: Sync site/ to S3
        run: aws s3 sync site/ "s3://${{ secrets.S3_BUCKET }}" --delete

      - name: Invalidate CloudFront cache
        run: >
          aws cloudfront create-invalidation
          --distribution-id "${{ secrets.CLOUDFRONT_DISTRIBUTION_ID }}"
          --paths "/*"
  • permissions: id-token: write — 冇呢句 OIDC 會失敗。
  • --delete — S3 上面已刪除嘅檔案會一齊刪走,S3 同 repo 保持一致。
  • concurrency — 短時間內連續 merge,舊嗰個 deploy 會被 cancel,唔會互相衝突。
  • Invalidation:每月首 1,000 個 path 免費,個人網站用 /* 最簡單。

驗證

  1. Merge 一個 PR 入 main(或者去 Actions tab 撳 Run workflow)。
  2. Actions log 兩步都綠色。
  3. 開 CloudFront domain(https://dxxxx.cloudfront.net)見到網站,改嘢再 merge 會更新。

常見問題

症狀原因 / 解法
開 root 見 403 AccessDeniedDefault root object 未設做 index.html,或者 bucket policy 未貼
Actions 報 Not authorized to perform sts:AssumeRoleWithWebIdentityTrust policy 個 sub 唔 match(repo 名 / branch 打錯),或者 workflow 冇 id-token: write
Deploy 成功但網站冇更新CloudFront cache 未 invalidate,check 埋 invalidation 嗰步有冇行
子目錄 /foo/ 開唔到 index.htmlDefault root object 只對 root 有效;需要嘅話加 CloudFront Function rewrite URI(本 site 全部用完整 .html path,唔受影響)
